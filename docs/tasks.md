# Rester タスクリスト

## セットアップ

- [ ] GitHub リポジトリの設定（GitHub Pages を有効化）
- [ ] `pnpm install` で依存関係をインストール・確認
- [ ] SolidStart の静的出力設定を確認（`prerender` / `crawlLinks: true`）
- [ ] GitHub Pages 向けベースパス（`base`）の設定

## UI / UX 設計

- [ ] ワイヤーフレーム作成（詩表示画面）
- [ ] フォント・タイポグラフィの選定（縦書き対応の検討）
- [ ] 「次へ」操作の仕様決定（ボタン / スワイプ / キーボード）
- [ ] カラースキーム・背景デザインの決定

## コンテンツ管理設計

- [ ] 詩データのスキーマ設計（JSON / YAML）
  - 必須フィールド: `id`, `body`, `author`
  - 任意フィールド: `title`, `year`, `source`
- [ ] データファイルの配置場所を決定（例: `app/src/data/poems.json`）
- [ ] サンプル詩データの作成（5〜10首、パブリックドメイン確認済み）
- [ ] パブリックドメイン確認基準の文書化

## 開発

- [ ] 詩表示コンポーネントの作成（本文 + 著者名）
- [ ] ランダム詩選択ロジックの実装（クライアントサイド）
- [ ] 一方向ナビゲーション（「次へ」のみ）の実装
- [ ] セッション内の表示済み詩の管理（重複防止）
- [ ] レスポンシブスタイルの適用
- [ ] ビルド確認（`pnpm build`）

## CI/CD（GitHub Actions）

- [ ] PR 時: コンテンツ形式バリデーション Workflow の作成
  - 詩データのスキーマ検証（必須フィールドの存在確認）
  - 著作権確認チェックリストの確認
- [ ] `main` ブランチへのマージ時: ビルド & GitHub Pages デプロイ Workflow の作成
- [ ] Workflow の動作確認（テスト PR での検証）

## コンテンツ投稿フロー整備

- [ ] `CONTRIBUTING.md` の作成（詩の追加手順・ルール）
- [ ] PR テンプレートの作成（著作権確認チェックリスト付き）
  - 著者の没年確認（没後70年以上）
  - 出典の記載
  - 本文の正確さ確認

## リリース

- [ ] GitHub Pages URL の確認・動作テスト
- [ ] `app/README.md` の更新（プロジェクト説明・公開URL）
- [ ] 初期コンテンツ（詩）の投入（最低10首）
- [ ] 公開アナウンス（任意）
