# Rester 要件定義

## プロジェクト概要

**Rester** は、没後70年以上経過した故人の詩・短歌をランダムに1首ずつ表示するWebサイトです。GitHub Pages上で静的サイトとして公開します。

---

## 目的・ゴール

- 故人の言葉を現代人に届ける
- 著作権が消滅したパブリックドメイン作品（没後70年以上）に限定し、合法的かつ持続可能なアーカイブを実現する

---

## ターゲットユーザー

- 詩・短歌に興味を持つ一般ユーザー
- 古典文学・近代文学に触れたいユーザー
- ランダムな出会いを楽しむユーザー

---

## 機能要件

### 表示

| # | 要件 |
|---|------|
| F-01 | 詩・短歌の本文を表示する |
| F-02 | 著者名を表示する |
| F-03 | 表示する詩・短歌はランダムに選択する |
| F-04 | ユーザーは「次へ」操作で次の詩・短歌に進める |
| F-05 | 前の詩・短歌に戻る操作は提供しない（一方向ナビゲーション） |

### コンテンツ管理

| # | 要件 |
|---|------|
| F-10 | 詩・短歌データは構造化ファイル（JSON または YAML）でリポジトリ内に管理する |
| F-11 | データスキーマ: `id`, `body`（本文）, `author`（著者名）, `title`（題名、任意）, `year`（作品年、任意）, `source`（出典、任意） |

---

## 非機能要件

| # | 要件 |
|---|------|
| N-01 | GitHub Pages での静的ホスティング |
| N-02 | SSG（静的サイト生成）により全ページをビルド時に生成する |
| N-03 | レスポンシブデザイン（モバイル・デスクトップ対応） |
| N-04 | UIは最小限に留める（詩の本文と著者名のみ表示） |
| N-05 | 高速な初回表示（静的ファイル配信による） |

---

## コンテンツポリシー

- **著作権**: 著作者の没年から70年以上経過した作品（パブリックドメイン）のみ掲載可
- **対象ジャンル**: 詩・短歌
- **言語**: 主に日本語（将来的に多言語対応も検討可）
- **確認義務**: 投稿者は著作権の消滅を確認してからPRを作成する

---

## 管理者フロー（コンテンツ追加）

```
1. 管理者がリポジトリをフォークまたはブランチを作成
2. 詩・短歌データファイルに新しい作品を追加
3. Pull Request を作成
4. GitHub Actions による形式バリデーション（自動）
5. レビュー・マージ
6. GitHub Actions により自動ビルド & GitHub Pages へデプロイ
```

---

## 制約・前提条件

- Node.js >= 22 が必要
- コンテンツはすべてリポジトリ内で管理する（外部DBなし）
- GitHub Pages の制約上、サーバーサイドの動的処理は行わない
- ランダム表示はクライアントサイドで実装する
